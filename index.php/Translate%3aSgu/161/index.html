Translate:Sgu/161

来自NOCOW
跳转到： 导航, 搜索

最近Vasya了解了一场在数学界及逻辑学界的有趣运动——“直觉逻辑”。这场运动的核心思想是否定排中律（一个命题非真即假）。Vasya很喜欢这个主意，所以他想在他所有的活动中（特别是科学研究中）使用“直觉逻辑”。但是这种逻辑比经典逻辑难得多。（这里居然有一个"di+cult"……）Vasya经常使用在经典逻辑中被允许但不被直觉逻辑允许的表达式。

现在他想写一个程序去帮助他检查表达式是否符合“直觉逻辑”。然后他“首先阅读了****的书籍。。进行了系统的学习”（wjb语）（Orz）。接着他就把这个任务交给你了……

这种逻辑的定义从一个有向无环图G=<V,E>开始。接着一个偏序定义在G上：对于G的两个顶点x,y，如果存在一条x到y的路径（长度可能为0），我们就称x<=y。

接着，我们考虑V的所有子集组成的集合B以及所有反链组成的集合H。（反链就是指V的一个子集，满足反链中任意两个不同元素都不能比较大小。注意反链包含空集与单点集。）

定义B到H的一个映射Max(M)（M属于B）。Max(M)是M的这样一个子集，满足对于Max(M)中的任何一个元素x，在M中都不存在一个元素y!=x使得x<=y。

接下来我们在H上定义一些运算。对于H的两个元素（即V的两个子集）a,b，定义：

a=>b 为b的一个子集，满足对于任何一个属于它的一个元素x，不存在一个a中的元素y使得x<=y；

a∧b = Max(a∪b)；

a∨b = Max({V的一个子集：对于它的每一个元素x，都存在a中的一个元素y与b中的一个元素z，满足x<=y且x<=z})；

0 = Max(V)；

1 = 空集；

~a = a=>0；

a==b = ((a=>b)∧(b=>a))。

现在考虑一个包含下面符号的逻辑表达式： 

       * 1与0；
       * 变量：从A到Z的大写字母；
       * 圆括号；
       * 否定 ~； 
       * 且 ∧，从左往右运算；
       * 或 ∨，依然从左往右运算；
       * 包含 =>，注意是从右往左运算；
       * 等价 ==，E1 == E2 == E3 == ... == En 相当于 (E1 == E2) ∧ (E2 == E3) ∧ ... ∧ (E(n-1) == En)。

上述各种符号按优先级从高到低给出。

一个表达式E被称作有效的当且仅当用H的任意一个子集替换掉E中的变量后，按照上面的规则计算结果都为1；否则该表达式就被称作无效的。

你的任务就是对于一个图G与一堆表达式，计算出哪些表达式是有效的，哪些是无效的。



[编辑] 输入文件

第一行包含两个空格隔开的数n,m ——图的结点数与边数(1<=n<=100,1<=m<=5000)。

接下来m行，每行两个整数si,ti，描述一条从si到ti的有向边。

接下来一行，包含一个整数k(1<=k<=20)，表示表达式个数。

接下来K行，每行一个表达式。表达式由0, 1, A, ..., Z, (, ), ~, &, |, =>, = 这些东西组成。其中后四种东西代表的是∧，∨，=>与==。可能有多余空格。表达式长度不超过254个字符，且 保证正确。

并且可以认为（……）H集合的元素个数不超过100个，并且所有你代入到表达式中的值的组数不会超过10的6次方。（就是让你直接暴力。）


[编辑] 输出文件

K行，每行一个"valid"或者"invalid"。

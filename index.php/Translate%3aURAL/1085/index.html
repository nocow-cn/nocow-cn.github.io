Meeting(聚会)

Time Limit: 5 second

Memory Limit: 1000K

[编辑] Discription

一群朋友们决定聚在一起，来庆祝他们在学院举办的学生竞赛中取得的胜利。不幸地，由于有轨电车票价上涨，出现了一个问题：因为所有人都分散居住在城市的各个地方，所以他们必须寻找一个合适的地点聚会，使他们所花的车费之和尽量少(他们要多留点钱来举行这次聚会)。你要帮助他们作出最佳选择。

所有车站由1到n的自然数编号，按顺序列出。城市里有m条有轨电车线(所有人只会乘坐有轨电车从一个车站到另一个车站，而决不步行)。每一条有轨电车线所经过的车站编号已经给出。假设有k个人。他们所拥有的钱数已经给出。我们也知道他们是否有月票。假定一张票为4卢布。

你要找一个车站，使所有人都可以到达那里且他们所花票钱之和最少。当然，他们可以转车。注意，转一次车要多买一张票。这些人们都很诚实——他们一定会买票。每个人用自己的钱买票。你不用考虑给他们留些钱来买返程票。

[编辑] Input Format

第一行包含两个自然数n、m，n<=100，m<=100(n——车站数，m——路线总数)。接下来的m行描述路线：在每行开头有一个数字L，表示路线经过车站总数(L<=100)。然后有L个数字，为路线经过的车站。这些数字分别由一个空格隔开。

紧接着的一行有一个自然数K，K<=100(为人数)，之后的k行为这些人的信息(一行描述一个人)：每行开头有一个非负整数，表示此人拥有的钱数(单位：卢布)。然后的一个数为此人从他家步行到达的车站(再从此车站乘车)。再然后是0(如果他没有月票)或1(如果他有月票)。以上的数字分别由一个空格隔开。任何人的钱都不大于1000卢布。

[编辑] Output Format

你要输出两个数：聚会地点的车站编号(如果有多个，输出编号最小的)和买票花费的钱之和(单位：卢布)。两个数字用一个空格隔开。如果人们无法聚会，则仅输出一个数0。

[编辑] Sample Input 

4 3
2 1 2
2 2 3
2 3 4
3
27 1 0
15 4 0
45 4 0

[编辑] Sample Output 

4 12

Problem Source: The 3rd high school children programming contest, USU, Ekaterinburg, Russia, March 4, 2001

汉诺塔


时间限制: 1 s 内存限制: 16 MB



背景


汉诺塔问题是一个著名的问题，描述如下： 有3根柱子，N个金属片依次串在第1根柱子上，编号大的在下，小的在上。 现在要把他们全部移到第2根柱子上。一次只能移动一片金属片。 所有移动过程中编号大的金属片不能叠在编号小的金属片上。 求至少要移几次，怎么移？


题目


汉诺塔问题的最优解可以用如下代码求得： 


procedure Hanoi(N:integer; From, To_, Temp : integer);
Begin
  if N>0 then
  begin
     Hanoi(N-1, From, Temp, To_);
     writeln (N, From, To_);
     Hanoi(N-1, Temp, To_, From)
  end
End;

步骤如图：

4 - 3 1, 2 3, 3, 2

7 - 1, 2, 3 - 2, 2, 2


意思就是，递归求解。如果只有1个金属片，直接移动就可以了。 如果有N个金属片要移，就要先把前N-1个先搬开到另外一个柱子上，然后把第N个移到目标柱上，最后把那N-1个也移到目标柱上。


给定N和某一时刻所有金属片所在的柱子的编号，求出这个时刻已经按照上面的方法移动了几步。


输入格式


第1行一个数N（1<=n<=31） 第2行N个数,每个数可能是1，2，3中一个，依次表示编号为1，2，3，4，5...N的金属片所在的柱子编号。


输出格式


一个数，移动到此状态所需的步数。 如果输入就是初始状态输出0。 如果你发现这个状态不是按照上面的移法得来的，输出-1。


样例输入 


3
3 3 1


样例输出


3
